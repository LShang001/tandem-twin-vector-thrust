### 1.2 验证方法四类定义

| 代码 | 方法 | 定义 | 适用场景 |
|------|------|------|----------|
| I | 检查 Inspection | 人工审查文档、代码、配置或页面元素，确认实现存在且符合约定 | 参数治理、接口契约、代码结构、DOM 元素 |
| A | 分析 Analysis | 通过数学推导、静态检查或代码走读确认属性（守恒量、符号、边界、稳定性） | 差速平方和不变、阻尼导数符号、子步等价性 |
| T | 测试 Test | 通过 `node --test` 自动化测试执行并断言 | 单元功能、回归基线、参数完整性 |
| D | 演示 Demonstration | 通过浏览器运行 Web 仿真并操作 / 观察结果 | 滑块响应、HUD 更新、演示序列、开关复位 |

## 3 模型可信度分级声明

| 可信度等级 | 结论 | 依据 |
|------------|------|------|
| **可用** | 控制律概念研究与通道解耦演示 | 46 项测试与 3 项回归基线覆盖所列工况；SAS 速率/姿态反馈极性、纵向配平参考量、横航向残差、零空速和完整复位已锁定 |
| **可用** | 教学与 layout 机理可视化 | Web 页面提供实时 6-DOF 动画与 HUD，方程面板解释模型来源；无飞行硬件风险 |
| **不可用** | 性能预测、包线保护设计、任何硬件决策 | 全部参数为 `MODEL-DEFAULT`（置信度 `low`），未经验证；执行机构为理想瞬时到位；气动为线性小迎角模型；无失速、无风扰、无传感器 / 电调 / 电池模型 |

任何将本仿真结果用于采购、制造、试飞或安全论证的行为，须先完成参数状态升级（台架测试、飞行辨识）、高保真气动与执行机构建模，并重新制定验证计划。

## 4 证据维护规则

### 4.1 行为变更

若修改以下任一行为保持红线，必须重新采集回归基线并在提交说明中记录原因：

- 帧上限 `frameCap = 0.05 s`、子步上限 `maxStep = 0.004 s`
- 电机时间常数 `tauM = 0.28 s`
- 差速开方分配公式
- SAS / 积分 / UI 限幅值
- 空速下限 `vMin = 0.5 m/s`
- 地面约束条件 `pos.z > 6.2 m && vWorld.z > 0`
- 四元数右乘 + 每步归一化
- 900 ms 自动演示触发
- 完整复位采用热配平转速，不模拟冷启动
- 渲染几何臂长 `±1.78 m` 与物理力臂 `0.62 m` 的区分
- `theta = -asin(R13)` 的欧拉提取约定

重采命令（在 `simulations/vector-thrust-lab` 目录）：

```bash
node tests/fixtures/generate.mjs
```

随后提交 `tests/fixtures/regression-baseline.json`，提交信息须以 `[regen-baseline]` 开头并说明变更动机。

### 4.2 参数变更

1. 修改 `models/aircraft-model.json`；
2. 运行 `py -3.12 tools/sync-params.py` 重新生成 `src/core/parameters.mjs`；
3. 运行 `py -3.12 tools/sync-params.py --check` 确认无漂移；
4. 更新本文档及 [`参数数据手册`](../registers/参数数据手册.md) 中的值、单位与状态字段；
5. 若参数变更导致行为变化，按 4.1 重采回归基线。

## 5 测试运行方法

在 `simulations/vector-thrust-lab` 目录执行：

```bash
node --test tests/
```

或显式列出测试文件：

```bash
node --test tests/math.test.mjs tests/parameters.test.mjs tests/control.test.mjs tests/propulsion.test.mjs tests/aerodynamics.test.mjs tests/dynamics.test.mjs tests/regression.test.mjs
```

参数漂移检测在仓库根目录执行：

```bash
py -3.12 tools/sync-params.py --check
```

浏览器 E2E 检查需先启动静态服务器：

```bash
cd simulations/vector-thrust-lab
python -m http.server 8080
```

然后在浏览器打开 `http://localhost:8080` 并按 2.4 节清单逐项检查。
