MOD-002 坐标系与符号约定

定位：代码 ↔︎ 文档的接口契约。凡阅读 simulations/vector-thrust-lab/src/core 源码或基于源码进行推导者，以本文档为符号与坐标约定准绳。本文档与 THY-001（纯理论推导）、MOD-001（整机动力学推导）互补，不重复推导过程。

1. 坐标系定义

坐标系 名称 原点 轴向 备注
{I} 惯性系 / 地面系 起飞点地面 x 北 / y 东 / z 向下（NED） 固定于地面；位置积分、地面约束在此系表达
{B} 机体系 质心 CG x_b 机头 / y_b 右翼 / z_b 向下 刚体固连；速度、角速度、力/力矩默认在此系
{M_f} / {M_t} 前 / 尾摆座系 前 / 尾电机中心 x_m 沿电机轴（推力方向），零摆角时与 x_b 重合 前座绕 z_b 摆动；尾座绕 y_b 摆动

坐标系关系图示：

1.1 关键方向约定

  • 重力：在 {I} 中表示为 { x:0, y:0, z:+g }，即 z 向下为正。
  • 高度：F.pos.z 为 NED 高度，数值增大代表离地更远；地面平面为 z = P.groundZ = 6.2 m。
  • 角度正方向：右手法则。绕 y_b 正转使机头下俯（标准航空俯仰正方向），但本仿真欧拉角提取对 theta 取负号，显示值与实际几何角符号相反（见第 4 节）。

2. 方向余弦矩阵（DCM）语义

2.1 矩阵定义

DCM R 实现机体基矢到惯性基矢的变换：

v^I = R · v^B

R 的第 j 列为 {B} 第 j 个基矢量在 {I} 中的坐标。R ∈ SO(3)，满足 R^T R = I、det R = +1。

2.2 列主序元素编号

代码中以 THREE.js / Matrix4 列主序访问旋转矩阵元素。e[k] 与 R_ij 的对应关系如下：

数组元素 数学元素 含义
e[0] R_11 机体系 x_b 在惯性系 x 轴投影
e[1] R_21 机体系 x_b 在惯性系 y 轴投影
e[2] R_31 机体系 x_b 在惯性系 z 轴投影
e[3] 0 齐次列
e[4] R_12 机体系 y_b 在惯性系 x 轴投影
e[5] R_22 机体系 y_b 在惯性系 y 轴投影
e[6] R_32 机体系 y_b 在惯性系 z 轴投影
e[7] 0 齐次列
e[8] R_13 机体系 z_b 在惯性系 x 轴投影
e[9] R_23 机体系 z_b 在惯性系 y 轴投影
e[10] R_33 机体系 z_b 在惯性系 z 轴投影
e[11] 0 齐次列

注：e[12..15] 为平移列（tx, ty, tz, 1），与姿态无关。

3. 四元数约定

3.1 分量顺序

代码中四元数表示为对象 { x, y, z, w }，对应数学记号：

q = (x, y, z, w) = q_x i + q_y j + q_z k + q_w

这与 THREE.Quaternion 的 (x, y, z, w) 分量顺序一致。

3.2 旋转方向

S.quat 表示 机体 → 惯性 的旋转。即对任意固连于机体的矢量 v^B，其在惯性系中的坐标为：

v^I = q ⊗ v ⊗ q*

其中 q* 为共轭，v 视为纯四元数 (0, v_x, v_y, v_z)。代码实现见 math.mjs:rotateVecByQuat，与 THREE.Vector3.applyQuaternion 公式一致。

3.3 逆与乘积

  • 单位四元数逆：q^{-1} = q* = (-x, -y, -z, w)，见 math.mjs:quatInvert。
  • 乘积 a ⊗ b 按 THREE.Quaternion.multiply 计算，见 math.mjs:quatMultiply。

3.4 更新方式

姿态积分采用显式欧拉、右乘机体系角速度：

Δq = ½ · q ⊗ (0, ω_x, ω_y, ω_z) · dt
q_{n+1} = normalize(q_n + Δq)

代码实现：dynamics.mjs:physicsStep 第 60–66 行。右乘是因为 ω 定义在机体系；若误用左乘，大姿态角下响应方向将错乱（理论依据见 THY-001 第 7 节）。

3.5 与浏览器层 THREE.Quaternion 的对应

S.quat 可直接赋值给 THREE.Quaternion(x, y, z, w) 用于渲染；telemetry.flight.quat 亦按此顺序输出给 HUD。

4. 欧拉角提取专节（锁定行为）

4.1 代码公式

math.mjs:eulerFromQuat 由单位四元数提取三个角度：

phi   = atan2(R_23, R_33)
theta = -asin(clamp(R_13, -1, 1))
psi   = atan2(R_12, R_11)

其中 R_ij 来自第 2.2 节列主序元素：R_13 = e[8]、R_23 = e[9]、R_33 = e[10]、R_12 = e[4]、R_11 = e[0]。

4.2 theta 的符号约定

上述公式对 theta 取了负号。这不是标准 3-2-1 欧拉角。其工程后果为：

  • 绕 +y_b 轴正向旋转 +a0（机头下俯）时，代码读出的 theta = -a0。
  • 标准航空约定下俯仰正方向应为 +theta；本代码中 HUD 显示的俯仰角与实际几何角符号相反。

示例（已固化在 tests/math.test.mjs）：

const a0 = 0.3; // rad
const q = { x:0, y:sin(a0/2), z:0, w:cos(a0/2) };
const e = eulerFromQuat(q);
// e.theta ≈ -0.3, e.phi ≈ 0, e.psi ≈ 0

4.3 为何保持锁定

该符号约定已深入以下环节，变更会改变闭环行为：

  1. SAS 增益整定：control.mjs:applySas 中俯仰角比例/积分反馈按 theta = F.euler.y 的负号整定；若将 eulerFromQuat 改为标准符号，SAS 对俯仰误差的响应极性将反转。
  2. 测试断言：tests/math.test.mjs 明确断言 e.theta ≈ -a0。
  3. HUD 显示：浏览器层直接显示 telemetry.flight.euler.y，飞行员/操作者已将负号纳入解读习惯。

因此，theta = -asin(R_13) 属于行为保持红线。任何修改必须走 GOV-002 变更流程，并同步重调 SAS、更新测试断言与 HUD 标注。

4.4 变更流程

如需修正为标准 3-2-1 约定：

  1. 在 math.mjs:eulerFromQuat 中移除 theta 负号；
  2. 在 control.mjs:applySas 中同步反转俯仰角/积分反馈符号；
  3. 更新 tests/math.test.mjs 断言；
  4. 在浏览器 HUD 中增加过渡标注，提示操作者符号已切换；
  5. 按 GOV-002 提交变更评审与回归测试。

5. 高度与地面基准

5.1 高度约定

  • NED 坐标系下 z 向下为正，故 F.pos.z 增大表示飞行器离地更远。
  • 起飞/重置时 F.pos.z = 0，即初始位置在地面上方 P.groundZ 处（见 state.mjs:resetFlightState）。

5.2 地面约束

地面平面位于惯性系 z = P.groundZ = 6.2 m（MODEL-DEFAULT）。触地逻辑在 dynamics.mjs:physicsStep 第 74–78 行：

if (F.pos.z > P.groundZ && F.vWorld.z > 0) {
    F.pos.z   = P.groundZ;
    F.vWorld.z = 0;
    F.vel = R^T(q) · F.vWorld;   // 将世界系速度逆变换回机体系
}

约束语义：

  • 仅当飞行器“钻入地面”（pos.z > groundZ）且仍在向下运动（vWorld.z > 0）时触发；
  • 钳位高度并清零下降速度；
  • 通过 rotateVecByQuat(vWorld, quatInvert(q)) 更新机体系速度，数值残差约 1e-6，测试用对应容差断言。

6. 渲染 / 物理比例声明

可视化模型与物理模型采用不同的长度比例，仅保证姿态与摆角语义一致：

项目 物理值（力学计算使用） 可视化值（browser 层） 来源
前电机力臂 a = 0.62 m 前机臂/电机位置 x = +1.78 models/aircraft-model.json / aircraft-view.mjs:182
尾电机力臂 b = 0.62 m 尾机臂/电机位置 x = -1.82 models/aircraft-model.json / aircraft-view.mjs:189

此外，cgRing、推力箭头、力矩箭头、坐标轴等视觉元素仅用于示意力/力矩方向与姿态，不保证几何尺度与物理量一一对应。任何基于截图或视觉尺寸的物理估算必须以 models/aircraft-model.json 为准。
