SAFE-001 初步安全计划

本篇为「纵列双发矢量推力飞行器」C0 概念仿真项目制定初步安全计划。范围限定在软件仿真本身：无飞行硬件、无人员直接暴露、无适航适用性；同时登记未来若向硬件化演进时应前瞻处理的安全议题。

1 范围声明

  • 项目形态：纯软件 Web 仿真与 Node 计算层，运行在开发者本地浏览器与测试环境中。
  • 物理风险：当前不存在飞行平台、动力系统、螺旋桨、高压电池或任何可运动机械部件，因此无直接人身伤害或财产损毁风险。
  • 适航与法规：本仿真不构成任何航空器、无人机系统或地面设备的设计数据，不适用 CCAR-91/92、FAR/CS 等适航或运行规章。
  • 安全计划目标：防止仿真结果被误用于超出其成熟度的工程决策；防止参数、模型与基线腐化导致的误导性输出。

2 软件层面风险与缓解

风险编号 风险描述 影响 当前缓解措施 残余风险 关联文档

3 未来硬件化前瞻议题

若本项目从概念仿真向台架测试、样机或飞行平台演进，以下议题须在专用安全计划中展开分析。当前仅作登记，不做展开：

  1. 螺旋桨破片与旋转部件防护：纵列双桨高速旋转，需评估桨叶失效动能与保护罩 / 安全距离。
  2. 锂电热失控与高压安全：飞行线若采用高功率电池，须制定过充 / 过放 / 短路 / 热蔓延预案。
  3. 失控保护与失效安全模式：单电机失效、通信中断、传感器失效时的安全降落 / 迫降策略。
  4. 围栏 / 系留与地面试车安全：系留点强度、地面试车区域隔离、紧急停机开关。
  5. 噪声与电磁兼容：大功率电机与电调对周边设备的电磁干扰及人员听力防护。
  6. 飞行包线与天气限制：基于真实参数重新建立包线，禁止在大风、结冰或低能见度下试飞。

4 与 CTL-002 及危害日志的接口

  • CTL-002 负责从控制分配角度分析单电机失效、执行机构饱和、姿态奇异等失效模式的可控性后果。
  • 危害与失效模式日志 负责维护仿真语境下的失效模式清单、严重度、检测手段与缓解方向。
  • 本安全计划仅覆盖软件仿真层面的风险；未来硬件安全计划应引用并扩展 CTL-002 与危害日志中的失效模式，补充 FMEA / FMECA 与定量风险评估。
