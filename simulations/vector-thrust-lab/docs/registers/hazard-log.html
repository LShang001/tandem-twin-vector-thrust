危害与失效模式日志

本日志登记「纵列双发矢量推力飞行器」C0 仿真项目已识别的失效模式与软件层面风险。严重度基于「纯软件仿真语境」评定：仅评估对仿真结论可信度、后续设计决策及概念研究的影响，不直接对应真实飞行事故等级。

编号 失效模式 原因 影响（仿真语境） 严重度 当前检测手段 缓解 / 改进方向 关联
HAZ-001 单前电机失效 前电机停转或失去拉力 失偏航主控；前电机反扭消失导致反扭失衡，产生持续滚转力矩 高 当前模型无电机失效逻辑，可通过手工构造 wf = 0 场景观察 增加电机失效保护逻辑；重新设计偏航 / 滚转应急分配；参考 CTL-002 可控性分析 CTL-002、ASM-003
HAZ-002 单尾电机失效 尾电机停转或失去拉力 失俯仰主控；尾电机反扭消失导致俯仰与滚转耦合扰动 高 当前模型无电机失效逻辑，可通过手工构造 wt = 0 场景观察 增加电机失效保护；评估尾电机失效后剩余俯仰控制力矩是否足够稳定进近 CTL-002
HAZ-003 差速大开度单机超 wMax 大滚转指令下 sqrt(1±Δω) 使一侧目标转速超过 wMax 模型仍输出推力，掩盖执行机构物理不可达；低油门侧可能先饱和，滚转效能骤降 中 代码审查与参数检查；当前无运行时告警 在 stepPropulsion 中增加转速限幅与差速重分配；在 UI 中提示差速可用范围 参数数据手册 PAR-004/038/039、ASM-014
HAZ-004 积分器持续饱和 持续姿态误差使 intTh / intPhi 达到上限 SAS 退出线性区，撤销误差后存在大超调与恢复延迟（windup） 中 control.test.mjs 验证积分限幅至 intThMax / intPhiMax 引入抗积分饱和（anti-windup）逻辑；结合执行机构限幅动态调整积分 CTL-001、tests/control.test.mjs
HAZ-005 姿态 ±90° 奇异 俯仰角接近 ±90° 时 3-2-1 欧拉角万向锁 欧拉角不连续，eulerFromQuat 输出突变；SAS 可能基于错误姿态角动作 中 math.test.mjs 锁定当前 theta = -asin(R13) 约定；大机动场景需人工观察 限制自动演示机动范围；或在控制律中改用四元数直接反馈 THY-001、ASM-010
HAZ-006 误导性结果被外推为真实性能或安全承诺 使用者忽略 MODEL-DEFAULT 与模型边界 导致未经验证的硬件、试飞或投资决策 高 成熟度声明见根 README、VER-001、CFG-000；参数手册标注置信度 强化文档与 UI 成熟度提示；使用前须签署/确认免责声明；禁止用于硬件决策直至参数升级 SAFE-001、VER-001
HAZ-007 参数误用 绕过 aircraft-model.json 直接修改 parameters.mjs；或使用未经验证的 JSON 仿真输入与受控文档不一致，结果不可复现 中 sync-params.py --check 漂移检测；生成文件头部警示 在 CI 中强制 --check；禁止提交手工修改的 parameters.mjs GOV-002、参数数据手册
HAZ-008 回归基线腐化 未记录原因即重新生成或手工篡改 baseline 行为变更被掩盖，回归测试失去意义 中 版本控制 diff；VER-001 规定重采须以 [regen-baseline] 提交并说明原因 强制代码审查；自动化检查 PR 是否含 baseline 变更说明 VER-001、SAFE-001

使用说明

  • 严重度仅用于概念仿真阶段的风险排序，不代表真实飞行安全等级。
  • 新增失效模式须经控制分配分析（CTL-002）与安全计划（SAFE-001）评审后按顺序编号加入。
  • 每条危害的缓解措施应转化为代码变更、测试用例或文档更新；闭环后在本日志中记录完成日期与证据。
