使用说明

  1. 每次新增试验前复制本模板，按顺序编号并填写完整。
  2. 「模型版本」「参数版本」「回归基线版本」应至少记录 Git commit hash，确保可追溯。
  3. 输入序列应尽量覆盖阶跃、脉冲、扫频、组合机动等典型模式；必要时引用 tests/scenarios.mjs 中的已有场景。
  4. 试验结论为「不通过」或「部分通过」时，必须填写后续行动并关联到具体需求、假设或危害。
